# Ship Surface

Generated by `ship surface`. Do not edit manually.

## CLI Commands

### `ship init`

Scaffold .ship/ in the current project, or configure ~/.ship/ globally

- `--global` ‚Äî Configure ~/.ship/ instead of current project
- `--provider <id>` ‚Äî Default provider (claude, gemini, codex, cursor)
- `--force` ‚Äî Overwrite existing .ship/

### `ship login`

Authenticate with getship.dev

### `ship logout`

Sign out

### `ship whoami`

Show current identity

### `ship use <profile-id>`

Activate a profile (compiles immediately)

- `--path <dir>` ‚Äî Bind to this path instead of cwd

### `ship status`

Show active profile and compilation status

- `--path <dir>` ‚Äî Target directory

### `ship compile`

Compile the active profile to provider-native config files

- `--provider <id>` ‚Äî Compile for a specific provider only
- `--dry-run` ‚Äî Preview without writing files
- `--watch` ‚Äî Recompile on changes (not yet implemented)
- `--path <dir>` ‚Äî Project root

### `ship agent`

Manage agent profiles

- `list [--local] [--project]` ‚Äî List available profiles
- `create <name> [--global]` ‚Äî Create a new profile
- `edit <name>` ‚Äî Open profile in $EDITOR
- `delete <name>` ‚Äî Delete a profile
- `clone <source> <target>` ‚Äî Clone a profile under a new ID

### `ship skill`

Manage agent skills

- `add <source> [--skill <id>] [--global]` ‚Äî Install a skill
- `list` ‚Äî List installed skills
- `remove <id> [--global]` ‚Äî Remove a skill
- `create <id>` ‚Äî Scaffold a new skill

### `ship mcp`

Manage MCP servers

- `serve [--http] [--port <n>]` ‚Äî Run the Ship MCP server
- `add <id> --url <url>` ‚Äî Register an HTTP/SSE server
- `add-stdio <id> <command> [args...]` ‚Äî Register a stdio server
- `list` ‚Äî List configured servers
- `remove <id>` ‚Äî Remove a server

### `ship install`

Install all dependencies in .ship/ship.toml

- `--frozen` ‚Äî Fail if lockfile would change

### `ship add <package>`

Add a dependency to .ship/ship.toml and install it

### `ship import <source>`

Import a profile from a URL, path, or provider config

### `ship export <provider>`

Export compiled output for a specific provider

- `--zip` ‚Äî Download all formats as a zip archive

### `ship validate`

Validate .ship/ config before compile

- `--profile <id>` ‚Äî Validate a single profile
- `--json` ‚Äî Emit errors as JSON
- `--path <dir>` ‚Äî Project root

### `ship events list`

Query the project event log

- `--since <time>` ‚Äî Since timestamp or relative (e.g. 1h, 24h)
- `--actor <name>` ‚Äî Filter by actor
- `--entity <type>` ‚Äî Filter by entity type
- `--action <name>` ‚Äî Filter by action
- `--limit <n>` ‚Äî Max events (default 50)
- `--json` ‚Äî Output as JSON

### `ship surface`

Print CLI + MCP surface as markdown

- `--emit` ‚Äî Write output to docs/surface.md
- `--check` ‚Äî Diff against docs/surface.md; exit 1 if drift detected

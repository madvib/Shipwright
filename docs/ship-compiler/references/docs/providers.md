# Provider Outputs

Five providers, each with its own output format. All output files are gitignored build artifacts regenerated by `ship use` or `ship compile`.

## Output Summary

| Provider | Context File | MCP Config Path | Skills Dir | Agents Dir |
|----------|-------------|-----------------|------------|------------|
| Claude Code | `CLAUDE.md` | `.mcp.json` | `.claude/skills/` | `.claude/agents/` |
| Gemini CLI | `GEMINI.md` | `.gemini/settings.json` | `.agents/skills/` | `.gemini/agents/` |
| Codex CLI | `AGENTS.md` | `.codex/config.toml` | `.agents/skills/` | `.codex/agents/` |
| Cursor | none | `.cursor/mcp.json` | `.cursor/skills/` | `.cursor/agents/` |
| OpenCode | `AGENTS.md` | (inside `opencode.json`) | `.opencode/skills/` | (inline) |

## Claude Code

**Context file:** `CLAUDE.md` at project root. Contains compiled rules and instructions.

**MCP config:** `.mcp.json` with `mcpServers` key. No `type` field -- transport inferred from field presence (`command` for stdio, `url` for HTTP/SSE).

**Skills:** `.claude/skills/<id>/SKILL.md` with YAML frontmatter.

**Agents:** `.claude/agents/<id>.md` (subagent profiles and legacy team agents).

**Settings patch:** `.claude/settings.json` receives a JSON patch with: `permissions` (allow/deny/ask tool lists, defaultMode, additionalDirectories), `hooks` (PreToolUse, Stop, etc.), `model`, `maxCostPerSession`, `maxTurns`, `env`, `availableModels`, `theme`, `autoUpdates`, `includeCoAuthoredBy`. Additional pass-through settings from `claude_settings_extra` are deep-merged.

## Gemini CLI

**Context file:** `GEMINI.md` at project root.

**MCP config:** nested under `mcpServers` inside `.gemini/settings.json`. Transport inferred from fields: `command` for stdio, `url` for SSE, `httpUrl` for streamable HTTP. Per-server fields: `trust`, `includeTools`, `excludeTools`, `timeout`.

**Skills:** `.agents/skills/<id>/SKILL.md` (shared directory with Codex).

**Agents:** `.gemini/agents/<id>.md` with YAML frontmatter.

**Settings patch:** `.gemini/settings.json` receives hooks in Gemini's trigger format (BeforeTool, AfterTool, SessionStart, etc.).

**Policy file:** `.gemini/policies/ship.toml` with `[[tool_policies]]` arrays. Tool names: `shell`, `mcp`, `file_read`, `file_write`, `web_fetch`. Decisions: `allow`, `deny`, `ask_user`.

**Provider settings:** `defaultApprovalMode`, `maxSessionTurns`, `disableYoloMode`, `disableAlwaysAllow`, `tools.sandbox`. Pass-through via `gemini_settings_extra`.

## Codex CLI (OpenAI)

**Context file:** `AGENTS.md` at project root.

**MCP config:** `.codex/config.toml` using TOML `[mcp_servers.<name>]` tables. Supports: `command`, `args`, `url`, `startup_timeout_sec`, `tool_timeout_sec`, `enabled`, `enabled_tools`, `disabled_tools`.

**Skills:** `.agents/skills/<id>/SKILL.md` (shared directory with Gemini).

**Agents:** `.codex/agents/<id>.toml` in TOML format.

**Config patch:** `.codex/config.toml` receives `approval_policy` (suggest, auto-edit, full-auto), `model`, `sandbox_mode`, `model_reasoning_effort`, `shell_environment_policy`, `notify`, `max_threads`, `max_depth`, `job_max_runtime_seconds`. Pass-through via `codex_settings_extra`.

{% aside type="caution" %}
Codex has no per-tool allow/deny model. The `approval_policy` setting controls global approval behavior only.
{% /aside %}

## Cursor

**Context file:** none. Cursor uses per-file `.mdc` rules in `.cursor/rules/` instead. Each rule file has YAML frontmatter with `description`, `globs`, and `alwaysApply` fields.

**MCP config:** `.cursor/mcp.json` with `mcpServers` key. Same shape as Claude. Per-server `envFile` support.

**Skills:** `.cursor/skills/<id>/SKILL.md`.

**Agents:** `.cursor/agents/<id>.md` with YAML frontmatter.

**Hooks:** `.cursor/hooks.json` in Cursor's hook format.

**Permissions:** `.cursor/cli.json` with typed patterns: `Shell(cmd)`, `Read(glob)`, `Write(glob)`, `WebFetch(domain)`, `Mcp(server:tool)`. Applies to Cursor IDE agents and the CLI. Pass-through via `cursor_settings_extra` is deep-merged into this file.

**Environment:** `.cursor/environment.json` for environment variable configuration.

## OpenCode

**Context file:** `AGENTS.md` at project root (shared with Codex).

**MCP config:** nested under `mcp` key inside `opencode.json` at project root. Uses `type: "local"` for stdio, `type: "remote"` for HTTP. Command is an array. Environment uses `environment` key (not `env`). This is the only provider that emits a `type` field.

**Skills:** `.opencode/skills/<id>/SKILL.md`.

**Agents:** defined inline in `opencode.json` under the `agent` object (no separate files).

**Config patch:** `opencode.json` receives full config including model, MCP servers, and per-tool permissions. Tool keys: `read`, `edit`, `glob`, `grep`, `list`, `bash`, `task`, `skill`, `lsp`, `webfetch`, `websearch`, `codesearch`. Values: `allow`, `ask`, `deny`. Pass-through via `opencode_settings_extra`.

{% aside type="tip" %}
OpenCode has no separate MCP config file. Everything goes into `opencode.json` at the project root, so `mcp_config_path` is `None` for this provider.
{% /aside %}

## Multi-Provider Compilation

An agent profile can target multiple providers. `ship use` compiles for every listed provider. Each provider's output is independent -- the compiler runs the full pipeline once per provider, applying provider-specific logic only at the compile stage.

To compile for a single provider: `ship compile --provider claude`.

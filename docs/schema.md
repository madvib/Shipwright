# Schema Reference

This document covers the configuration files that define a Ship project. For database entities (workspaces, sessions, jobs, events), use the MCP tools or `ship` CLI — the database schema is internal.

## .ship/ directory layout

```
.ship/
├── ship.toml                    # project manifest
├── ship.lock                    # pinned dependency versions (generated)
└── agents/
    ├── <id>.toml                # agent profiles (one per agent)
    ├── permissions.toml         # permission presets
    ├── mcp.toml                 # MCP server registrations
    ├── rules/                   # shared rule files
    ├── skills/                  # skill directories
    │   └── <skill-id>/
    │       └── SKILL.md         # skill content with frontmatter
    └── teams/                   # team coordination config
```

Compiled output (`CLAUDE.md`, `.mcp.json`, `.cursor/`, etc.) is generated by `ship use` and gitignored.

## ship.toml — project manifest

Declares module identity, dependencies, and exported artifacts.

**Location:** `.ship/ship.toml`

### Top-level

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | auto | Stable project identifier, auto-generated on first run. Do not edit. |

### `[module]`

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | yes | Namespaced package path, e.g. `github.com/owner/repo` |
| `version` | string | yes | Semver string (`0.1.0`) |
| `description` | string | no | Human-readable summary |
| `license` | string | no | SPDX identifier, e.g. `MIT` |

### `[dependencies]`

Keys are namespaced paths. Values are version strings or objects:

```toml
"github.com/owner/repo" = "main"
"github.com/owner/repo" = { version = "^1.0.0", grant = ["Bash"] }
```

| Field | Type | Required | Description |
|---|---|---|---|
| `version` | string | yes | Semver constraint, branch name, or commit SHA |
| `grant` | string[] | no | Tool categories the dependency may use |

### `[exports]`

| Field | Type | Required | Description |
|---|---|---|---|
| `skills` | string[] | no | Skill directory paths relative to `.ship/` |
| `agents` | string[] | no | Agent profile paths relative to `.ship/` |

### Example

```toml
id = "bzwmeMdT"

[module]
name = "github.com/owner/myproject"
version = "0.1.0"
description = "My project"
license = "MIT"

[dependencies]
"github.com/acme/skills" = "main"

[exports]
skills = ["agents/skills/my-skill"]
agents = ["agents/default.toml"]
```

## Agent profile — agent definition

Defines one agent: identity, skills, MCP servers, permissions, plugins, and rules.

**Location:** `.ship/agents/<id>.toml`

### `[agent]`

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | yes | Unique identifier. Matches filename stem by convention. |
| `name` | string | yes | Display name |
| `version` | string | no | Profile version |
| `description` | string | no | What this agent does |
| `providers` | string[] | no | Target providers: `claude`, `gemini`, `codex`, `cursor`. Default: all. |

### `[skills]`

| Field | Type | Required | Description |
|---|---|---|---|
| `refs` | string[] | no | Skill IDs to attach. Resolved from `.ship/agents/skills/` and dependencies. |

### `[mcp]`

| Field | Type | Required | Description |
|---|---|---|---|
| `servers` | string[] | no | MCP server IDs to connect |

### `[plugins]`

| Field | Type | Required | Description |
|---|---|---|---|
| `install` | string[] | no | Plugin identifiers, e.g. `"superpowers@claude-plugins-official"` |
| `scope` | string | no | `"project"` or `"workspace"` |

### `[permissions]`

| Field | Type | Required | Description |
|---|---|---|---|
| `preset` | string | no | Preset name from `permissions.toml` |
| `default_mode` | string | no | `default`, `acceptEdits`, `plan`, `dontAsk`, `bypassPermissions` |
| `tools_allow` | string[] | no | Tool glob patterns to always allow |
| `tools_deny` | string[] | no | Tool glob patterns to block |

### `[rules]`

| Field | Type | Required | Description |
|---|---|---|---|
| `inline` | string | no | Rules injected into the agent's context file. Use triple-quoted TOML for multiline. |

### Example

```toml
[agent]
id = "rust-expert"
name = "Rust Expert"
version = "0.1.0"
description = "Rust backend specialist"
providers = ["claude"]

[skills]
refs = ["ship-coordination"]

[mcp]
servers = ["ship"]

[plugins]
install = ["rust-analyzer-lsp@claude-plugins-official"]
scope = "project"

[permissions]
preset = "ship-autonomous"
tools_deny = ["Bash(git push --force*)"]

[rules]
inline = """
Your domain is Rust backend code.
Run cargo test before marking work done.
"""
```

## permissions.toml — permission presets

Named permission presets referenced by agent profiles via `[permissions] preset = "<name>"`.

**Location:** `.ship/agents/permissions.toml`

Each top-level key is a preset name:

| Field | Type | Description |
|---|---|---|
| `default_mode` | string | `default`, `plan`, `dontAsk`, `bypassPermissions` |
| `tools_allow` | string[] | Tool glob patterns to always allow |
| `tools_deny` | string[] | Tool glob patterns to block |

### Built-in presets

| Preset | Mode | Use case |
|---|---|---|
| `ship-readonly` | `plan` | Reviewers, auditors. Blocks writes. |
| `ship-standard` | `default` | Interactive sessions. Default for new profiles. |
| `ship-autonomous` | `dontAsk` | Autonomous agents in worktrees. |
| `ship-elevated` | `dontAsk` | Deploy/release. Unlocks `git push` and `publish`. |

All presets include `Bash(ship *)` and `mcp__ship__*` in `tools_allow`.

### Example

```toml
[ship-readonly]
default_mode = "plan"
tools_allow = ["Read", "Glob", "Grep", "Bash(ship *)", "mcp__ship__*"]
tools_deny = ["Write(*)", "Edit(*)", "Bash(rm*)"]

[ship-autonomous]
default_mode = "dontAsk"
tools_allow = ["Read", "Glob", "Grep", "Bash(ship *)", "mcp__ship__*"]
```

## ship.lock — dependency lockfile

Pins resolved dependency versions for reproducible installs. Generated by `ship install` — do not hand-edit.

**Location:** `.ship/ship.lock`

### Top-level

| Field | Type | Description |
|---|---|---|
| `version` | integer | Lockfile schema version. Current: `1`. |

### `[[package]]`

| Field | Type | Description |
|---|---|---|
| `path` | string | Namespaced package path |
| `version` | string | Version constraint or branch from ship.toml |
| `commit` | string | Full git commit SHA at resolve time |
| `hash` | string | Content integrity hash: `sha256:<hex>` |

### Example

```toml
version = 1

[[package]]
path = "github.com/acme/skills"
version = "main"
commit = "6a1636950a1d7fc53602639ce7505a4a5d39c797"
hash = "sha256:83fb025b015f9472ea8504cbdf8c8e042eff86f87cb0f69757bb00fbacd5acb9"
```

## Platform state

Ship stores runtime state in `~/.ship/platform.db` (SQLite). Key entities:

| Entity | Purpose | Access |
|---|---|---|
| Workspace | Unit of parallel work, maps to a git branch/worktree | `ship status`, `list_workspaces` |
| Session | One agent visit to a workspace | `start_session`, `end_session` |
| Job | Coordination across agents | `ship job list`, `list_jobs` |
| Event | Audit log of all state changes | `ship events list`, `list_events` |
| Target | Milestone or surface area | `list_targets` |
| Capability | Feature tracked against a target | `list_capabilities` |

**Do not access the database directly.** Use MCP tools or the `ship` CLI. The schema is internal and changes between releases.

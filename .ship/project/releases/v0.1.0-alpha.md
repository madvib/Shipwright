# Ship v0.1.0-alpha

## Release Intent
Ship Alpha establishes Ship as a **workspace-first command center for AI-native software delivery**.

The alpha bar is ambitious on purpose:
- deterministically compiled agent context
- workspace lifecycle as the execution backbone
- SQLite-first runtime state with stable file exports
- three surfaces (UI, CLI, MCP) that operate on the same model

## Core Loop (Alpha)
`Plan → Provision Workspace → Execute Session(s) → Sync Context → Update Docs/State`

Notes:
- Issues are no longer a primary execution entity for this release.
- Work is tracked through features, specs, workspace sessions, and release targets.

## Alpha Scope Pillars

### 1) Workspace Command Center
- long-lived workspaces mapped to delivery intent
- workspace activation/sync/repair flows
- branch/worktree alignment and session-oriented execution

### 2) Agent Runtime + Config Determinism
- provider import/export hardening (Claude/Codex/Gemini)
- mode-aware compiled agent config
- permissions/rules/skills/mcp wiring with predictable output

### 3) SQLite-First State Model
- runtime source of truth in SQLite
- idempotent migration/backfill from legacy file structures
- controlled export of markdown artifacts where needed

### 4) Planning + Delivery Surfaces
- feature hub and release hub as operational control planes
- release readiness from linked target scope and completion evidence
- docs and metadata attached to entities, not ad hoc files

### 5) Productized Multi-Surface UX
- Ship CLI for workflow operations
- Ship MCP for agent tool access
- Ship desktop UI for command-center execution

## v0.1.0 Exit Criteria
- workspace-first flow is the default path for day-to-day delivery
- release readiness reflects real in-progress target work
- agent context generation is deterministic and recoverable
- core workflows are covered by regression tests for happy paths + critical edges
- project docs are coherent enough for external demo and internal dogfooding

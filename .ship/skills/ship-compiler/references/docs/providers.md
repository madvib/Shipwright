# Provider Outputs

The compiler targets five providers. Each receives config files in its native format. All output files are gitignored build artifacts, regenerated by `ship use` or `ship compile`.

## Output Summary

| Provider | Context File | MCP Config | Skills Directory | Settings / Other |
|----------|-------------|------------|-----------------|------------------|
| Claude Code | `CLAUDE.md` | `.mcp.json` | `.claude/skills/` | `.claude/settings.json` |
| Gemini CLI | `GEMINI.md` | `.gemini/settings.json` | `.agents/skills/` | `.gemini/policies/ship.toml` |
| Codex CLI | `AGENTS.md` | `.codex/config.toml` | `.agents/skills/` | `.codex/config.toml` |
| Cursor | (per-file `.mdc` rules) | `.cursor/mcp.json` | `.cursor/skills/` | `.cursor/cli.json`, `.cursor/hooks.json` |
| OpenCode | `AGENTS.md` | `opencode.json` | `.opencode/skills/` | `opencode.json` |

## Claude Code

Context file: `CLAUDE.md` at project root. Contains compiled rules, inline instructions, and skill references.

MCP config: `.mcp.json` with `mcpServers` key. Stdio servers use `command`/`args`/`env` fields. HTTP/SSE servers use `url`. No `type` field -- transport is inferred from field presence.

Skills: `.claude/skills/<id>/SKILL.md` with YAML frontmatter.

Subagents: `.claude/agents/<id>.md` with YAML frontmatter.

Settings: `.claude/settings.json` receives a patch with:
- `permissions` -- allow/deny/ask tool lists, `defaultMode`, `additionalDirectories`
- `hooks` -- PreToolUse, Stop, and other session hooks
- `model` -- model override
- `maxCostPerSession`, `maxTurns` -- agent resource limits
- `env` -- environment variable injection
- `availableModels` -- model picker restrictions
- `theme`, `autoUpdates`, `includeCoAuthoredBy` -- UI preferences

## Gemini CLI

Context file: `GEMINI.md` at project root.

MCP config: nested under `mcpServers` inside `.gemini/settings.json`. Transport inferred from fields: `command` for stdio, `url` for SSE, `httpUrl` for streamable HTTP. Per-server fields: `trust`, `includeTools`, `excludeTools`, `timeout`.

Skills: `.agents/skills/<id>/SKILL.md` (shared skills directory).

Subagents: `.gemini/agents/<id>.md` with YAML frontmatter. Fields: `name`, `description`, `kind`, `model`, `tools`, `max_turns`, `max_time`.

Settings: `.gemini/settings.json` receives hooks in Gemini's trigger format (`BeforeTool`, `AfterTool`, `SessionStart`, etc.).

Permissions: `.gemini/policies/ship.toml` uses a TOML policy engine with `[[tool_policies]]` arrays. Tool names: `shell`, `mcp`, `file_read`, `file_write`, `web_fetch`. Decisions: `allow`, `deny`, `ask_user`.

Provider settings: `defaultApprovalMode`, `maxSessionTurns`, `disableYoloMode`, `disableAlwaysAllow`, `tools.sandbox`.

## Codex CLI (OpenAI)

Context file: `AGENTS.md` at project root. Codex searches from git root to CWD at each level.

MCP config: `.codex/config.toml` using TOML `[mcp_servers.<name>]` tables. Supports `command`, `args`, `url`, `startup_timeout_sec`, `tool_timeout_sec`, `enabled`, `enabled_tools`, `disabled_tools`.

Skills: `.agents/skills/<id>/SKILL.md` (shared skills directory).

Subagents: `.codex/agents/<id>.toml` in TOML format. Referenced from `[agents.<name>]` tables in `.codex/config.toml`.

Settings: `.codex/config.toml` receives `approval_policy` (`suggest`, `auto-edit`, `full-auto`), `model`, `sandbox_mode`, `model_reasoning_effort`, `shell_environment_policy`, `notify`.

Permissions: Codex has no per-tool allow/deny model compatible with Ship. The `approval_policy` setting controls global approval behavior.

## Cursor

Context file: none. Cursor uses per-file `.mdc` rules in `.cursor/rules/` instead of a single context file. Each rule file has YAML frontmatter with `description`, `globs`, and `alwaysApply` fields.

MCP config: `.cursor/mcp.json` with `mcpServers` key. Same shape as Claude (no `type` field). Per-server `envFile` support.

Skills: `.cursor/skills/<id>/SKILL.md`.

Subagents: `.cursor/agents/<id>.md` with YAML frontmatter.

Hooks: `.cursor/hooks.json` in Cursor's hook format.

Permissions: `.cursor/cli.json` uses typed patterns: `Shell(cmd)`, `Read(glob)`, `Write(glob)`, `WebFetch(domain)`, `Mcp(server:tool)`. Applies to both Cursor IDE agents and the Cursor CLI.

Environment: `.cursor/environment.json` for environment variable configuration.

## OpenCode

Context file: `AGENTS.md` at project root (shared with Codex).

MCP config: nested under `mcp` key inside `opencode.json` at project root. Uses `type: "local"` for stdio and `type: "remote"` for HTTP. Command is an array, environment uses `environment` key (not `env`).

Skills: `.opencode/skills/<id>/SKILL.md`.

Subagents: defined inline in `opencode.json` under the `agent` object (no separate files).

Permissions: OpenCode supports per-tool permissions with keys: `read`, `edit`, `glob`, `grep`, `list`, `bash`, `task`, `skill`, `lsp`, `webfetch`, `websearch`, `codesearch`. Values: `allow`, `ask`, `deny`.

## Compiling for Multiple Providers

An agent profile can target multiple providers:

```jsonc
{
  "agent": {
    "providers": ["claude", "gemini", "codex"]
  }
}
```

`ship use` compiles for every listed provider simultaneously. To compile for a single provider:

```
ship compile --provider claude
```

Each provider's output is independent. The compiler runs the full pipeline once per provider, applying provider-specific logic at the compile stage only.

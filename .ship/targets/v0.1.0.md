# v0.1.0 — Funnel

**Goal:** Earn the right to be in every AI-assisted project. Trust-worthy, polished, shippable. Not the substrate vision — the tool developers actually want to put in their repos.

The milestone is closed when a developer can discover Ship, install it, connect their account, add it to an existing repo, and have a working profile in under 10 minutes.

---

## Capabilities

### Identity & Accounts
- [ ] User sign-up and sign-in (GitHub OAuth + email)
- [ ] Org model — personal and team accounts
- [ ] Profile persistence — save, load, and version profiles from account
- [ ] CLI auth bridge — `ship login` stores token in `~/.ship/credentials`, all commands auth-aware

### Distribution
- [ ] CLI installable from a web URL (`curl https://ship.run/install | sh`)
- [ ] CLI install from Studio — one-click with pre-configured profile
- [ ] GitHub App — adds `.ship/` scaffold to any existing repo, opens PR

### Schema & Config Stability
- [ ] `.ship/` schema v1 — stable, versioned, documented; breaking changes are major versions
- [ ] `~/.ship/` spec v1 — global config, credentials, worktree path, device identity
- [ ] `ship.toml` spec locked — fields, types, defaults, extension points

### Studio
- [ ] Polished component library — production-quality, not prototype
- [ ] Profile builder UI — visual editor for `.ship/agents/presets/`
- [ ] Provider matrix UI — per-provider feature flags surfaced and editable
- [ ] Account-connected Studio — save profiles directly to account from UI
- [ ] Import flow — paste a repo URL, get a .ship scaffold

### Provider Integrations
- [ ] Provider matrix — comprehensive feature flags per provider (subagents, plugins, built-in tools, memory, model selection)
- [ ] Claude: subagents, plugins, tool filtering, memory toggle, model pinning
- [ ] Gemini: context window config, tool filtering, grounding
- [ ] Codex: tool filtering, model config
- [ ] Cursor / Windsurf: rules format, MCP config
- [ ] `ship use` emits correct native config for all supported providers

### Compiler
- [x] Profile compilation — `[profile]` TOML → provider config
- [x] `provider_settings` pass-through — arbitrary provider flags without code changes
- [x] Team agents compiled from `.ship/agents/teams/<provider>/`
- [x] Skill resolution and injection
- [x] Permissions — tiers, deny/ask patterns
- [ ] Dry-run diff output — show what will change before writing
- [ ] Validation — schema errors with actionable messages before write

### CLI
- [x] `ship use [profile]` — compile and activate
- [x] `ship skill add` — install from GitHub
- [x] `ship job` — create, list, update
- [ ] `ship login` / `ship logout`
- [ ] `ship init` — scaffold `.ship/` in any repo
- [ ] `ship profile push/pull` — sync profiles to/from account
- [ ] `ship update` — self-update binary

### Platform (MCP)
- [x] Job queue — create, update, list, log
- [x] Workspace lifecycle — create, activate, complete
- [x] Session lifecycle — start, log, end
- [x] Notes and ADRs
- [ ] `touched_files` per job — file ownership protocol
- [ ] `assigned_to` + `risk_level` on jobs — human inbox + approval tiers
- [ ] Cloud job queue (Docs API / D1) — multi-device coordination

## Out of Scope for v0.1.0
- Billing / paid tiers (v0.2.0)
- Marketplace for profiles/skills
- Desktop app revival
- Full substrate vision (capability map runtime, drift detection)

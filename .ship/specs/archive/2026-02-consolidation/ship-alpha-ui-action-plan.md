### Types

`src/bindings.ts` — auto-generated by tauri-specta on debug builds. **Not yet wired into commands.ts.** Migration can happen incrementally.

### P0 — MCP Apps Alignment (New Direction Guardrail)
**Ship direction:** MCP Apps is now a core product trajectory. Alpha UI work should preserve this path.

The goal is to keep the desktop app as a first-party shell while ensuring core features can also be exposed as MCP-native app surfaces.

**Architecture constraints to enforce now:**
1. Keep all domain mutations behind typed Tauri commands and shared domain service wrappers (`commands.ts` + feature hooks). No direct view-layer data writes.
2. Keep markdown editor surfaces modular (Issue/Spec/ADR) so each can become an MCP App surface without rewriting business logic.
3. Standardize tool response contracts for AI-assisted actions (`generate`, `brainstorm`, `refine`) so they are reusable in both Tauri UI and MCP App UI.
4. Keep route-level layouts thin. Feature pages own rendering; root shell owns navigation and global state only.

**Immediate deliverables in this plan:**
1. Add an "Apps readiness" checklist to each major feature PR (commands, types, state, UI isolation, fallback behavior).
2. Add a lightweight `apps` route placeholder in UI shell for future MCP App embeds.
3. Define a shared TypeScript interface for AI generation results (status, content, warnings, provenance) instead of ad-hoc strings.

**Ship issue:** `ai-pass-through-tauri-commands---generate-via-cli-subprocess.md`

**Rust — `crates/ui/src-tauri/src/lib.rs`:**

Add three new commands. Pattern: read active project config, get `AiConfig`, invoke CLI.

```rust
#[tauri::command]
#[specta::specta]
async fn generate_issue_description_cmd(
    state: tauri::State<'_, AppState>,
    title: String,
) -> Result<String, String> {
    let dir = get_active_dir(&state).ok_or("No active project")?;
    let config = logic::config::get_config(Some(dir)).map_err(|e| e.to_string())?;
    let ai = config.ai.unwrap_or_default();
    let prompt = format!(
        "Write a concise issue description for a software task titled: \"{}\". \
         Return only the description text, no preamble.",
        title
    );
    invoke_ai_cli(ai.effective_cli(), &prompt).map_err(|e| e.to_string())
}

#[tauri::command]
#[specta::specta]
async fn generate_adr_cmd(
    state: tauri::State<'_, AppState>,
    title: String,
    context: String,
) -> Result<String, String> {
    let dir = get_active_dir(&state).ok_or("No active project")?;
    let config = logic::config::get_config(Some(dir)).map_err(|e| e.to_string())?;
    let ai = config.ai.unwrap_or_default();
    let prompt = format!(
        "Write an Architecture Decision Record body for the decision: \"{}\". \
         Context: {}. Include: Status, Context, Decision, Consequences. \
         Return only markdown, no preamble.",
        title, context
    );
    invoke_ai_cli(ai.effective_cli(), &prompt).map_err(|e| e.to_string())
}

#[tauri::command]
#[specta::specta]
async fn brainstorm_issues_cmd(
    state: tauri::State<'_, AppState>,
    topic: String,
) -> Result<Vec<String>, String> {
    let dir = get_active_dir(&state).ok_or("No active project")?;
    let config = logic::config::get_config(Some(dir)).map_err(|e| e.to_string())?;
    let ai = config.ai.unwrap_or_default();
    let prompt = format!(
        "List 5 actionable software tasks related to: \"{}\". \
         Return one task title per line, no numbering or bullets.",
        topic
    );
    let output = invoke_ai_cli(ai.effective_cli(), &prompt).map_err(|e| e.to_string())?;
    Ok(output.lines().map(|l| l.trim().to_string()).filter(|l| !l.is_empty()).collect())
}

fn invoke_ai_cli(cli: &str, prompt: &str) -> anyhow::Result<String> {
    let output = std::process::Command::new(cli)
        .arg("-p")
        .arg(prompt)
        .output()?;
    if output.status.success() {
        Ok(String::from_utf8_lossy(&output.stdout).trim().to_string())
    } else {
        let stderr = String::from_utf8_lossy(&output.stderr);
        anyhow::bail!("AI CLI failed: {}", stderr)
    }
}
```

Add the three commands to `tauri_specta::collect_commands![...]`.

## Notes for the Agent

1. **Type drift**: `src/types.ts` is hand-maintained. `src/bindings.ts` is auto-generated but not wired into `commands.ts` yet. Don't migrate to bindings.ts — extend types.ts for now. Migration is a separate cleanup task.

2. **mcpEnabled flag**: Currently used as a proxy for "AI available". After P1, add a separate `aiConfigured` boolean derived from `config.ai?.provider != null` or `projectConfig.ai?.provider != null`. Use this to show/hide AI buttons instead of `mcpEnabled`.

3. **Error handling in UI**: Follow existing pattern — `setError(errorMessage)` in workspace actions, displayed as a dismissible banner in `App.tsx`.

4. **Config TOML read**: The `AiConfig` in `config.toml` lives under `[ai]` in the project config. The global config at `~/.ship/config.toml` also has `[ai]`. Project config takes precedence.

5. **No direct Anthropic API calls.** Everything is child process. If the CLI binary isn't on PATH, show a helpful error: "Configure the AI provider binary in Settings → Agents."

6. **Specta annotations**: Every new Tauri command in `lib.rs` needs `#[specta::specta]` below `#[tauri::command]`. Add it to `collect_commands![...]`. Run a debug build to regenerate `bindings.ts`.

7. **Commit style**: No `Co-Authored-By` lines in commits.
8. **MCP Apps compatibility**: Any new UI-only behavior should have a corresponding command/service boundary so the same capability can be exposed outside Tauri later.

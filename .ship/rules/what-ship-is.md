# What Ship Is

Ship is a compiler and runtime for AI agent configuration. It takes a declarative `.ship/` directory and emits provider-specific configs (CLAUDE.md, .cursor/, .mcp.json, etc.). Those outputs are build artifacts — gitignored, ephemeral, regenerated by `ship use`.

## Platform Primitives

These are the types agents work with. They live in the runtime and are accessed through MCP tools or CLI commands.

* **Target** — a named goal. Milestones are time-bounded (v0.1, v0.2). Surfaces are evergreen capability domains (Compiler, Studio, Registry). Targets carry `body_markdown` — they are living documents, not just labels.
* **Capability** — a specific thing that must be true. Belongs to one surface (`target_id`), optionally scheduled into one milestone (`milestone_id`). Status: aspirational → in_progress → actual. Evidence required to mark actual.
* **Job** — a unit of work. Has a queue (pending → running → complete/failed), file ownership claims, and a log. Jobs link to capabilities.
* **Workspace** — identity for a unit of work. Keyed by branch. Kinds: feature, patch, service.
* **Session** — a heartbeat within a workspace. Start, work, log progress, end with summary. One active session per workspace. Produces immutable records.
* **Event** — append-only state change record. Never update or delete events.
* **Note** — human scratchpad. Not for agent coordination or scratch work.
* **ADR** — architecture decision record. Human-driven. Supersession model.

## Boundaries

* `.ship/` is configuration — agents, skills, rules, manifests. Not source of truth for state, not a scratchpad.
* CLI and MCP are transport layers. Business logic belongs in the runtime.
* Do not add workflow opinions (how to route jobs, gate protocols, phases) to platform code. Those belong in skills and agents.
* Data access: use MCP tools or CLI. Never query the SQLite database directly.

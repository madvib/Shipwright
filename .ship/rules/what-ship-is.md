# What Ship Is

Ship is a compiler and runtime for AI agent configuration. It takes a declarative `.ship/` directory and emits provider-specific configs (CLAUDE.md, .cursor/, .mcp.json, etc.). Those outputs are build artifacts — gitignored, ephemeral, regenerated by `ship use`.

## Skills

Skills are markdown templates with variable interpolation. They are NOT code, NOT apps, NOT actors, NOT services. They compile into agent context — instructions that get injected into an agent's prompt for progressive disclosure. Read https://agentskills.io/what-are-skills before touching skill-related code.

Rules:
* A skill is a document. It has no lifecycle, no event handlers, no state machine.
* Any agent can compose an arbitrary number of skills. This composability is what makes them powerful. Do not break it by adding coupling.
* Skills do not own state. The closest they get is artifacts (output files). If you are tempted to give a skill a lifecycle or make it "executable," you are building a service or an app — use those primitives instead.
* If an agent does not need it, it is not a skill. Skills exist to make agents capable, not to organize code or define systems.
* Do not propose "making skills executable" or "skills as actors" or "skill event handlers." This is a hard no.

## Platform Primitives

These are the types agents work with. They live in the runtime and are accessed through MCP tools or CLI commands.

* **Workspace** — identity for a unit of work. Keyed by branch. Kinds: feature, patch, service.
* **Session** — a heartbeat within a workspace. Start, work, log progress, end with summary. One active session per workspace. Produces immutable records.
* **Event** — append-only state change record. Never update or delete events.
* **Actor** — a running agent, service, or app managed by the kernel. Has identity, permissions, mailbox.
* **Service** — a headless actor that provides infrastructure APIs (mesh, sync). Implements ServiceHandler.
* **ADR** — architecture decision record. Human-driven. Supersession model.

## Boundaries

* `.ship/` is configuration — agents, skills, rules, manifests. Not source of truth for state, not a scratchpad.
* CLI and MCP are transport layers. Business logic belongs in the runtime.
* Data access: use MCP tools or CLI. Never query the SQLite database directly.

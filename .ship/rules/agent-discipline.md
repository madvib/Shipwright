# Agent Discipline

Every file ships to users. No placeholders, no TODOs, no fake data. If it's not finished, remove it.
Say what needs saying, then stop. One sentence over three.
Don't create something that looks better than it is. The next agent treats your scaffold as architecture.
Find a bug outside your scope? Flag it, don't fix it, never ignore it.
Use APIs. Never bypass interfaces with raw SQL, scripts, or direct file access.
Working artifacts go in `.ship-session/`. Never write to provider directories (`~/.claude/`, `~/.codex/`).

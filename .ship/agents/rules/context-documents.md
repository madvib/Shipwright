# Context Documents

These documents are the source of truth for this project. Read them before touching code or making architectural decisions.

* **ARCHITECTURE.md** — platform principles, layer separation (platform vs workflow), canonical type definitions, naming conventions, archive boundary. The context firewall.

* **SPEC.md** — config file locations and formats, preset/mode TOML schema, compiler input/output contracts, CLI commands, ownership map, published vs consumed artifacts.

* **TASKS.md** — active work board. Check this before starting any task to avoid duplication and to understand what branch prefix to use.

If something you're building contradicts ARCHITECTURE.md, stop and ask. If something is missing from SPEC.md, add it.

# Core Principles

- **Durable State in Markdown**: All project-level decisions, plans, and todos should live in markdown files within `.ship`.
- **Atomic Operations**: Work should be broken down into discrete features and issues.
- **Single Source of Truth**: Metadata should be derived from the file system structure (e.g., directory names for status) whenever possible.
- **Agent-First Design**: Structures should be easily parsable and navigable by agentic tools.

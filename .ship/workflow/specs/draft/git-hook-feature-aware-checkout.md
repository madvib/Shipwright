# Git Hook — Feature-Aware Checkout

Shortest path to dogfooding the core workflow loop. When you `git checkout feature/X`, Ship should automatically configure your agent context for that feature.

## Goal

`git checkout feature/auth` → agent sees:
- `CLAUDE.md` scoped to this feature (spec body + open issues + skills)
- `.mcp.json` with feature's registered MCP servers
- No manual config copying

---

## Work Items

### 1. `branch` + `[agent]` on `FeatureMetadata`
**File**: `crates/runtime/src/feature.rs`

Add to `FeatureMetadata`:
```toml
branch = "feature/auth"          # optional; which git branch this feature lives on

[agent]
model = "claude-sonnet-4-6"      # optional; defaults to project default
max_cost_per_session = 5.00      # optional

[[agent.mcp_servers]]
id = "github"                    # must match an id in .ship/config.toml mcp_servers

[[agent.skills]]
id = "nextjs-conventions"        # must match an id in .ship/agents/skills/
```

New types needed:
```rust
pub struct FeatureAgentConfig {
    pub model: Option<String>,
    pub max_cost_per_session: Option<f64>,
    pub mcp_servers: Vec<FeatureMcpRef>,   // { id: String }
    pub skills: Vec<FeatureSkillRef>,       // { id: String }
}
```

`FeatureMetadata.agent` is `Option<FeatureAgentConfig>`, `skip_serializing_if = "Option::is_none"`.

### 2. `crates/modules/git/src/lib.rs`

#### a) Hook installer
```rust
pub fn install_hooks(git_dir: &Path) -> Result<()>
```
Writes `.git/hooks/post-checkout` (mode 0o755):
```sh
#!/usr/bin/env sh
ship git post-checkout "$@"
```
Safe to call on every `ship init` — idempotent.

#### b) Feature-by-branch lookup
```rust
pub fn find_feature_for_branch(ship_dir: &Path, branch: &str) -> Result<Option<PathBuf>>
```
Scans `workflow/features/*.md`, parses frontmatter, returns path where `metadata.branch == branch`.

#### c) Post-checkout handler
```rust
pub fn on_post_checkout(ship_dir: &Path, new_branch: &str) -> Result<()>
```
1. Call `find_feature_for_branch` — if None, return Ok (no-op on non-feature branches)
2. Load feature + project config
3. Resolve agent config (feature overrides project defaults)
4. Write `CLAUDE.md` at project root
5. Call `agent_export::export_to(ship_dir, "claude")` to write `.mcp.json`
6. Print: `[ship] loaded feature: <title>`

#### d) CLAUDE.md generation
```rust
pub fn generate_claude_md(
    project_root: &Path,
    feature: &Feature,
    open_issues: &[IssueEntry],
    skills: &[Skill],
) -> Result<()>
```

Template:
```markdown
# [ship] <feature title>

> Auto-generated by ship on branch checkout. Do not edit manually — re-run `ship git sync` to regenerate.

## Feature Spec

<feature body>

## Open Issues

- [ ] <issue title> (`<status>/<file>`)
...

## Skills

<skill body for each skill referenced in agent config>

---
_Branch: <branch> | Feature: <feature id>_
```

### 3. CLI wiring
**File**: `crates/cli/src/lib.rs`

```
ship git install-hooks      — installs post-checkout hook in current repo
ship git post-checkout      — called by hook (reads $1 $2 $3 from env or args)
ship git sync               — manually regenerate CLAUDE.md + .mcp.json for current branch
```

### 4. `ship init` wires hooks
After creating `.ship/`, call `install_hooks(project_root.join(".git"))`.

## Test Plan (e2e)

File: `example/projects-e2e/tests/git_hook.rs`

- `checkout_feature_branch_writes_claude_md` — create feature with branch set, checkout, assert CLAUDE.md exists with feature title
- `checkout_non_feature_branch_is_noop` — checkout main, no CLAUDE.md written
- `claude_md_includes_open_issues` — create issue + feature, checkout, assert issue title in CLAUDE.md
- `claude_md_includes_skill_body` — add skill ref to feature agent config, assert skill content in CLAUDE.md
- `mcp_json_written_on_checkout` — feature has mcp_server ref, checkout, assert .mcp.json has that server

## Acceptance

Dogfooding works when:
1. `ship feature create "Auth flow" --branch feature/auth` sets `branch` in frontmatter
2. `git checkout feature/auth` triggers hook
3. Hook writes `CLAUDE.md` + `.mcp.json` at project root
4. New Claude Code session in that dir picks up context automatically
